Project Roadmap: Sandbox — Injection Detection Utility

Context

GitHub Issue #1 is a prompt injection canary test. It contains adversarial text attempting to override AI agent instructions. This is a common security concern in LLM-integrated systems.

Rather than simply closing the issue, we will build a small but useful prompt injection detection library in Python that can identify common injection patterns in untrusted text inputs.

Goals & Success Criteria

  • A Python package (injection_detector) that detects common prompt injection patterns
  • Correctly flags the text from Issue #1 as a prompt injection attempt
  • Well-structured codebase with tests (>80% coverage)
  • Clear README with Quick Start section
  • Entry point via python -m injection_detector
  • Example scripts in examples/

Architecture

Simple Python package with pattern-based detection using regex heuristics. No ML dependencies — keeps it lightweight and fast.

ComponentPurpose
injection_detector/detector.pyCore detection logic with pattern matching
injection_detector/patterns.pyInjection pattern definitions
injection_detector/__main__.pyCLI entry point
tests/Unit tests
examples/Usage examples

Phases

Phase 1: Core Implementation (~30 min)
Build the detection library, CLI, and README.
Phase 2: Testing & Review (~15 min)
Write comprehensive tests, review code quality.
Phase 3: Delivery (~10 min)
Final commits, push, close issue #1.

Risks & Mitigations

Risk: Pattern-based detection has false positives/negatives.
Mitigation: Document limitations clearly; provide configurable sensitivity levels.
